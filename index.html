Java Multithreading

1. Introduction to Concurrency

Concurrency allows a program to perform multiple operations simultaneously, maximizing CPU utilization and improving application responsiveness.

üéØ The Human Analogy

Much like humans can walk, talk, and breathe simultaneously, computers can download files, print documents, and run a system clock in parallel. This natural parallelism is what multithreading aims to achieve in software.

Process vs. Thread

üì¶ Process

  • Heavyweight - Significant resource overhead
  • Dedicated Memory - Each process has its own memory space
  • Isolated - Processes don't share memory directly
  • Slow Communication - Requires IPC mechanisms

üßµ Thread

  • Lightweight - Minimal resource overhead
  • Shared Memory - Threads share the same memory space
  • Efficient Communication - Direct memory access
  • Fast Creation - Quick to create and destroy
üìä Process vs Thread Memory Layout

üéØ The "Why" of Multithreading

Multithreading gives Java powerful capabilities not found in natively single-threaded languages like C or C++. A primary use case is streaming media, where one thread downloads data while another plays it simultaneously to ensure a smooth user experience.

3. Thread States: Life Cycle of a Thread

A thread exists in various states throughout its execution.

üîÑ Thread Life Cycle State Diagram

State Descriptions:

  • Born: Thread created but not started
  • Ready: Waiting for processor time
  • Running: Actively executing
  • Blocked: Waiting for I/O or monitor lock
  • Sleeping: Temporarily paused
  • Waiting: Waiting for notification
  • Dead: Execution completed

5. Thread Synchronization

When multiple threads share the same data, they can interfere with each other. Synchronization ensures data integrity.

public class Counter {
    private int count = 0;
    
    // Synchronized method
    public synchronized void increment() {
        count++;
    }
    
    // Wait and notify example
    public synchronized void waitForValue(int target) throws InterruptedException {
        while (count < target) {
            wait(); // Releases the lock
        }
    }
    
    public synchronized void notifyThreads() {
        notifyAll(); // Wakes up all waiting threads
    }
}
Synchronized Block Visualization

üîÑ Wait/Notify Mechanism

  • wait(): Causes thread to release the lock and wait
  • notify(): Wakes up a single waiting thread
  • notifyAll(): Wakes up all waiting threads

6. Daemon Threads

A Daemon thread is a "service provider" thread that runs in the background.

üëª Daemon vs User Threads
// Creating a daemon thread
Thread daemonThread = new Thread(() -> {
    while (true) {
        System.out.println("Daemon running...");
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            break;
        }
    }
});

daemonThread.setDaemon(true); // Must be called before start()
daemonThread.start();
‚ö†Ô∏è Important: The JVM exits when only daemon threads are running. Daemon threads are terminated abruptly when the last user thread completes.

7. Runnable Interface

While you can create a thread by extending the Thread class, implementing the Runnable interface is often preferred.

‚ùå Extending Thread

class MyThread extends Thread {
    public void run() {
        System.out.println("Running");
    }
}

// Usage
new MyThread().start();

‚úÖ Implementing Runnable

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Running");
    }
}

// Usage
new Thread(new MyRunnable()).start();
Runnable Pattern

üéØ Why Runnable is Preferred:

  • Flexibility: Your class can extend another class while still being runnable
  • Separation of Concerns: Separates task (Runnable) from executor (Thread)
  • Reusability: Same Runnable can be used with different executors
  • Better OOP Design: Follows composition over inheritance principle

8. Thread Groups

Thread Groups allow you to categorize threads into a single object for easier management.

üå≥ Thread Group Hierarchy
// Creating thread groups
ThreadGroup parentGroup = new ThreadGroup("ParentGroup");
ThreadGroup childGroup = new ThreadGroup(parentGroup, "ChildGroup");

// Creating threads in groups
Thread t1 = new Thread(parentGroup, () -> {
    System.out.println("Thread in parent group");
});

Thread t2 = new Thread(childGroup, () -> {
    System.out.println("Thread in child group");
});

// Managing entire group
parentGroup.interrupt(); // Interrupts all threads in group
parentGroup.setMaxPriority(Thread.MAX_PRIORITY); // Sets max priority for all

üìä Thread Group Benefits:

  • Bulk Operations: Interrupt or modify priority of entire group at once
  • Organization: Creates tree-like hierarchy for better control
  • Security: Can restrict access to threads within a group
  • Monitoring: Easy to enumerate and monitor related threads
